## Кратко

Семантическая вёрстка — особый подход к написанию HTML-разметки страниц. При этом подходе разработчик опирается не на содержание сайта, а на смысловое предназначение каждого блока и логическую структуру HTML-страницы.

## Семантические теги

В современном стандарте HTML есть большое количество семантических тегов под разные логические блоки веб-документа. Например:

- [`<article>`](/html/article/) — тег для создания самостоятельных блоков контента. Его содержимое может быть карточкой статьи в блоге, карточкой товара, рекламным баннером и любым другим самодостаточным логическим блоком.
- [`<aside>`](/html/aside/) — тег для дополнительного контента, используется для создания «сайдбара» или бокового меню на сайтах.
- [`<nav>`](/html/nav/) — контейнер для навигации со ссылками на другие страницы и разделы сайта.
- [`<section>`](/html/section/) — логический контейнер для объединия содержимого по смыслу. Его часто путают с `<article>`. `<section>` используют для отделения одной смысловой части от другой, а `<article>` — самодостаточный контейнер со своим содержимым.
- [`<main>`](/html/main/) — контейнер для основного содержимого страницы.
- [`<header>`](/html/header/) — тег для выделения вводного содержимого или навигации. Не привязан к конкретному месту страницы или внутрии её отдельной секции, но традиционно используется для создания основной навигации по сайту — «шапки». Также можно использовать для оборачивания оглавления раздела, заголовка статьи с вводной информацией о ней и т. д.
- [`<footer>`](/html/footer/) — тег с информацией о разделе. Например, это могут быть данные о том, когда была написана статья, её авторе и т. д. Тоже не имеет жёсткой привязки к положению на странице или отдельной секции, к которой относится.

К примеру, здесь у `<article>` есть дата начала старта курса, которую можно поместить в «подвал» карточки:

```html
<article>
  <header>
    <h2>SEO для начинающих</h2>
    <span class="author">Иван Иванович</span>
  </header>

  <p>
    Благодаря этому курсу вы научитесь задавливать конкурентов
    с помощью ссылочной массы, а не качественного контента.
  </p>

  <footer>
    <time datetime="2023-10-23">
      23 октября 2023
    </time>
  </footer>
</article>
```

## Как семантически разметить страницу?

На самом деле всё просто. Разберём по шагам.

**Первый шаг**. Вы смотрите на крупные блоки всего сайта и выделяете их под теги для крупных блоков: `<header>`, `<footer>`, `<main>`, `<nav>`, `<aside>`. Обычно эти блоки повторяются на всех страницах.

![На карточке две области выделены прямоугольными фигурами. В обе вписаны четыре других прямоугольных разноцветных блока. Слева прямоугольники подписаны тегами div с классами header, sidebar, content и footer. Справа в прямоугольнике правильная семантическая вёрстка, где div были заменены на теги header, nav, main и footer, у которых нет классов.](images/example-semantic-markup.jpg)
Пример семантической структуры сайта.

**Второй шаг**. Далее вы смотрите на содержимое страницы. Исходя из него размечаете заголовки [`<h1>`-`<h6>`](/html/h1-h6/). Общепринято делать всего один заголовок первого уровня `<h1>` на странице. Также помните о правильном порядке заголовков.

Разберём на примере. На странице всего один заголовок первого уровня. Заголовки третьего уровня находятся в том же блоке, что и заголовок `<h2>`, являясь его подзаголовками. Третьей секцией снова идёт `<h2>` уже со своим содержимым.

```html
<main>
  <section>
    <h1></h1>
    <p></p>
  </section>

  <section>
    <h2></h2>
    <p></p>

    <h3></h3>
    <p></p>

    <h3></h3>
    <p></p>
  </section>

  <section>
    <h2></h2>
    <p></p>
  </section>
</main>
```

**Третий шаг**. Обращаем внимание на небольшие смысловые разделы страницы. Списки, таблицы, примеры кода, параграфы текста, цитаты и похожие элементы оборачиваются в свои теги.

**Четвёртый шаг**. Разбираемся с самыми мелкими элементами: кнопками, временем, изображениями, ссылками, видео и т. д.

## Почему семантика важна?

Рассмотрим причины, зачем вообще использовать семантическую вёрстку.

- **SEO**. Семантическая вёрстка важна для поисковой оптимизации сайта. Это связано с тем, что поисковые движки не могут понимать сам текст, его контекст и то, насколько он важен. Благодаря использованию семантических тегов в вёрстке помогаем поисковой системе понять содержимое сайта и проиндексировать его относительно других сайтов со схожей тематикой.
- **Доступность**. Зрячим пользователям не составит проблем визуально понять, какой блок они видят на основе его внешнего вида. Слабовидящие и незрячие пользователи используют [скринридеры](/a11y/screenreaders) для чтения информации со страницы. Благодаря правильной семантической вёрстке улучшаем их пользовательский опыт. Также существует проблема «кнопок». До сих пор люди часто используют `<div>` и стилизуют его визуально, забывая о нативных браузерных событиях: фокусе, нажатию по <kbd>Enter</kbd> и пробелу. Люди, которые пользуются клавиатурой для навигации по сайту, не смогут нажать такую «кнопку».
- **Читаемость исходного кода**. При использовании семантических тегов легче поддерживать исходный код проекта в будущем. Куда удобнее ориентироваться в вёрстке, если она построена на семантических тегах.
