## Кратко

Добавить стили на страницу можно по-разному. Можно определить стили глобально, разместив их в HTML, подключить внешний CSS-файл, определить стили с помощью JavaScript, либо стилизовать элемент отдельно, используя внутренние стили.

Среди разработчиков хорошим тоном считается писать стили в отдельном файле и затем подключать их с помощью тега [`<link>`](/html/link). А вот внутренних стилей стараются избегать, хотя их применение также бывает полезно.

## Почему это важно?

То, как вы реализуете подключение стилей на вашу страницу, с одной стороны позволяет определить, как быстро отобразится ваша страница, с другой - может привести к неожиданным последствиям. Для этого необходимо понимать, какие достоинства и недостатки несёт каждый из способов подключения, и почему конкретный способ может не подойти.


## Подключение стилей из внешнего файла

При использовании такого подхода описание CSS-свойств производится в отдельном файле с расширением _.css_. Такие стили ещё называют связанными.

Для связывания документа с этим файлом применяется тег `<link>`, который помещается в [`<head>`](/html/head):

```html
<head>
  <link rel="stylesheet" href="main.css">
</head>
```

Атрибуты `rel="stylesheet"` указывает, что данный файл является таблицей стиля в формате CSS. В качестве значения `href` указывается путь к CSS-файлу. Он может быть задан как относительно, так и абсолютно.
Таким образом можно подключать таблицу стилей, которая находится на другом сайте. В примере выше мы подключаем шрифт Roboto с сайта [Google Fonts](https://fonts.google.com).

А вот какие стили содержатся в файле _main.css_:

```css
h1 {
  color: green;
  font-size: 24px;
}

p {
  font-size: 24px;
}
```


### Преимущества

- Возможность использования одного CSS-файла для стилизации на нескольких страницах. При изменении стиля в одном единственном файле, стиль автоматически применяется ко всем страницам, где есть на него ссылка.
- При первой загрузке файл со стилем помещается в кэш браузера, и загрузка страницы происходит быстрее.
- Внешние стили позволяют гибко использовать [псевдоклассы](/css/pseudoclasses) и [псевдоэлементы](/css/pseudoelements). К примеру разработчик может задать интерактивные состояния отдельно выбранным кнопкам, стилизовать каждый чётный элемент списка или выполнить другие задачи.

Внешний CSS также открывает доступ к [кастомным свойствам](/css/custom-properties).

Этот способ стилизации позволяет создавать переменные и с их помощью гибко управлять стилями. Например, быстро менять цвета:

```css
:root {
  --main-bg-color: #2196f3;
  --white: #fff;
  --black: #000000;

  --blue-lighten: #64b5f6;
  --blue-darken: #1565c0;

  --purple: #9c27b0;
  --purple-light: #4a148c;
  --purple-depp: #ce93d8;

  --grey: #808080;
  --grey-lighten: #e0e0e0;
  --grey-darken: #424242;
}

body {
  background-color: var(--main-bg-color);
}

h1 {
  color: var(--main-text);
}

p {
  color: var(--grey-darken)
}
```

## Подключение глобальных стилей

В данном случае CSS-свойства описываются в самом HTML-документе внутри тега [`<style>`](/html/style) и располагаются в шапке страницы:

```html
<head>
  <style>
    h1 {
      color: tomato;
    }

    p {
      font-size: 24px;
    }
  </style>
</head>
```

### Преимущества

- Поскольку код добавляется непосредственно в HTML-файл, браузеру нет необходимости загружать сторонние файлы, что позволяет отрисовать страницу быстрее.
- Глобальные стили работают изолированно и применяются непосредственно к странице, на которой прописаны. В таком подходе можно смело использовать классы или идентификаторы, не задумываясь о проблемах [каскада](/css/cascade).

### Недостатки:

- С каждым новым правилом вес HTML-файла будет увеличиваться. Это приводит к росту потребляемого трафика и уменьшению скорости загрузки страницы.
- Со временем может возникнуть потребность использовать аналогичные стили на других страницах. В таком случае CSS придётся дублировать.

## Подключение внутренних стилей

В таком случае элементам на странице, которые необходимо стилизовать, добавляется атрибут [`style`](/html/global-attrs/#style). Значением этого атрибута могут быть свойства и значения CSS, которые будут применены к данному элементу:

```html
<h1 style="сolor: yellow; margin: 10px 20px">Заголовок</h1>
<p style="text-align: right; font-size: 12px">Текст</p>
```

Стили, заданные через атрибут `style` также называют _инлайн-стилями_.

### Преимущества данного подхода:

- В случае с инлайн-стилями, по аналогии с глобальными, браузеру нет необходимости делать дополнительный запрос на сервер, чтобы загрузить файл стилей.
- Встроенные стили легко и быстро добавлять. Не нужно беспокоиться о написании правильного селектора, поскольку они добавляются непосредственно к элементу, который вы хотите изменить.
- Такие стили бывают удобны, если стилизация задана с помощью сторонних библиотек или через CMS. Это удобно для тестирования, предварительного просмотра изменений или для выполнения быстрых исправлений.
- Данный подход может помочь написать сайт в максимально короткие сроки. Это может быть удобно, если сайт небольшой, а времени на его написание не особо много.

## Почему инлайн-стили — это плохо?

### Приоритет

Внутренние стили имеют [наивысший приоритет](https://doka.guide/css/specificity/#atribut-style), их нельзя переопределить с помощью селекторов по [идентификатору](/css/id-selector), [классу](/css/class-selector) или [тегу](/css/tag-selector) во внешнем CSS. Единственный способ это сделать — добавить [`!important`](/css/important) к значению свойства. Но этот приём тоже считается плохой практикой.

### Переиспользование

При использовании инлайн-стилей мы зачастую будем вынуждены добавлять встроенный стиль к каждому элементу, который необходимо стилизовать. В том случае, если элементов на странице много, прописывать стили вручную станет неудобно. К тому же это нарушает принцип «разделения содержимого и оформления».

### Псевдоклассы и псевдоэлементы

Ещё одним недостатком встроенных стилей является невозможность определить и стилизовать [псевдоклассы](/css/pseudoclasses) и [псевдоэлементы](/css/pseudoelements).
Атрибут `style` определяет свойства стиля для конкретного HTML-элемента. Псевдоклассы и псевдоэлементы в атрибуте не встречаются, поэтому нельзя определить и стилизовать их встроенно.

### Восприятие кода

Если писать стили внутри атрибута `style`, то HTML становится трудночитаемым. Логическая структура исчезает и стили размываются по всему коду. И чем крупнее проект, тем сложнее управлять стилизацией.

Согласитесь, что такой код читать намного труднее:

```html
<h1 style="сolor: yellow; margin: 10px 20px 15px 10px;">Заголовок</h1>
<div style="border: 1px solid red; max-widht: 500px; margin: auto">
  <p>У моей кошки <span style="color: lightblue;">светло синие</span> глаза.</p>
</div>
```

Нежели этот:

```html
<h1 class="title">Заголовок</h1>
<div class="wrapper">
  <p>У моей кошки <span class="eyes">светло синие</span> глаза.</p>
</div>
```

## Импорт CSS

При использовании данного метода происходит подключение стороннего файла, аналогично с подключением внешних стилей.
Основное отличие данного метода в том, что импорт позволяет подключить одну таблицу стилей в другую.
В таком случае содержимое стороннего файла импортируется с помощью команды [`@import`](/css/import).

### Преимущества

По мере того как ваш проект растёт в размерах и сложности, поддержка CSS-файлов, может стать трудной. `@import` помогает разбить массивную таблицу на более понятные и логичные части.

Чтобы проиллюстрировать это, достаточно сделать ссылку на CSS-файл через тег `<link>`, например:

```html
<link href="/styles/main.css" rel="stylesheet" type="text/css">
```

Затем эта таблица стилей ссылается уже на несколько других вспомогательных таблиц:

```css
@import url("/styles/header.css");
@import url("/styles/navbar.css");
@import url("/styles/footer.css");
@import url("/styles/grid.css");
```

### Недостатки

- При нормальных обстоятельствах загрузка файлов CSS с использованием директивы `@import` может значительно увеличить время загрузки страницы, поскольку CSS-файлы ссылаются один за другим. Как правило, `@import` увеличивает количество соединений с сервером, поскольку файл, на который идёт ссылка, также нужно скачать и проанализировать.
- Данный способ не рекомендуется использовать, так как он не гарантирует последовательную загрузку файлов CSS.
