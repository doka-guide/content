## Кратко

Для изменения [контекста выполнения](/js/function-context/) функции используется один из трех методов:

- `bind()` — возвращает новую функцию с привязанным [`this`](/js/function-context/);
- `call()` — вызывается сразу, аргументы через запятую;
- `apply()` — вызывается сразу, аргументы массивом.

## Bind

Создаёт новую функцию с указанным [`this`](/js/function-context/) и (опционально) аргументами.

### Как пишется

```js
const newFunc = func.bind(context, arg1, arg2, ...)
```

### Как понять

Возвращает новую функцию. Первый аргумент становится [`this`](/js/function-context/) внутри функции - и его нельзя изменить повторным вызовом `bind()`.

```js
const cat = { name: 'Барсик' };
const dog = { name: 'Шарик' };

function sayHi() {
  console.log(`Привет, я ${this.name}`);
}

// Привязываем к коту
const bound = sayHi.bind(cat);
bound(); // "Привет, я Барсик"

// Пытаемся переопределить контекст
bound.call(dog);  // "Привет, я Барсик" — не изменилось!
bound.apply(dog); // "Привет, я Барсик" — не изменилось!

const boundAgain = bound.bind(dog);
boundAgain();     // "Привет, я Барсик" — даже двойной bind не помог!
```

Все остальные аргументы передаются в функцию по порядку, причем можно передать только часть аргументов (по порядку), а остальное добавить при вызове новой функции. Пример:

```js
const player = { name: 'Артур' };

function attack(enemy, damage) {
  console.log(`${this.name} атакует ${enemy} на ${damage} урона`);
}

const arthurAttack = attack.bind(player, 'дракона'); // фиксируем enemy = 'дракона'

arthurAttack(100); // "Артур атакует дракона на 100 урона"
arthurAttack(150); // "Артур атакует дракона на 150 урона"
```

### Когда использовать

- при передаче функции в [setTimeout](/js/settimeout/)/[setInterval](/js/setinterval/)/[addEventListener](/js/element-addeventlistener/), чтобы она не потеряла контекст выполнения;
- фиксация [`this`](/js/function-context/) в функции;
- создание функции с предустановленными аргументами, [каррирование](/tools/fp/#karrirovanie).

## Call

Вызывает функцию с указанным [`this`](/js/function-context/) и аргументами по порядку.

### Как пишется

```js
func.call(context, arg1, arg2, ...)
```
### Как понять

Вызывает функцию сразу. Первый аргумент становится [`this`](/js/function-context/) внутри функции, а все остальные аргументы передаются в функцию по порядку.

### Когда использовать

Для одиночного вызова функции с конкретным [`this`](/js/function-context/), если аргументы лежат в отдельных переменных, а не в массиве.

## Apply

Вызывает функцию с указанным [`this`](/js/function-context/) и аргументами в виде массива.

### Как пишется

```js
func.apply(context, [arg1, arg2, ...])
```

### Как понять

Вызывает функцию сразу. Первый аргумент становится [`this`](/js/function-context/) внутри функции, а второй (массив) — её аргументами.

### Когда использовать

Для одиночного вызова функции с конкретным [`this`](/js/function-context/), если аргументы уже в массиве.

## Сравнение методов

Каждый из методов по-свому решает задачу привязки [`this`](/js/function-context/) функции:

| Метод       | Вызывается   | Возвращает       | Передача аргументов      |
|-------------|--------------|------------------|--------------------------|
| `call`      | сразу        | результат функции| по одному, через запятую |
| `apply`     | сразу        | результат функции| массивом                 |
| `bind`      | позже        | новую функцию    | по одному (можно частями)|

Пример со всеми тремя методами:

```js
const player = { name: 'Артур' };

function attack(enemy, damage) {
  console.log(`${this.name} атакует ${enemy} на ${damage} урона`);
}

// call — сразу, аргументы через запятую
attack.call(player, 'гоблина', 30); // "Артур атакует гоблина на 30 урона"

// apply — сразу, аргументы массивом
attack.apply(player, ['орка', 45]); // "Артур атакует орка на 45 урона"

// bind — возвращает новую функцию
const arthurAttack = attack.bind(player, 'дракона');
arthurAttack(100); // "Артур атакует дракона на 100 урона"
arthurAttack(150); // "Артур атакует дракона на 150 урона"
```
