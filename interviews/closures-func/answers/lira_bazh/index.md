Замыкание — это комбинация функции и лексического окружения, в котором эта функция была объявлена. Простыми словами: это когда внутренняя функция «запоминает» переменные из внешней функции, даже после того как внешняя функция завершила свою работу.

Главная задача замыкания - хранить значения переменных внутри функции, чтобы они не лежали в глобальном контексте, но к ним можно было получить доступ. Замкнутые переменные никак нельзя изменить, кроме как дочерними функциями.

Базовым примером замыкания является счётчик:

```js
function createCounter() {
    // переменная во внешней функции
    let count = 0;

    return function() {
      // внутренняя функция изменяет значение count и возвращает его
      count++;
      return count;
    };
}

const counter = createCounter();
console.log(counter());
// 1
console.log(counter());
// 2
console.log(counter());
// 3
```

Таким образом, напрямую изменить значение `count` нельзя, это возможно только посредством использования функции `counter()`. Это и называется замыканием.

Подробнее о замыканиях можно прочитать в [статье об области видимости](/js/closures/#zamykanie).
